# IViewFor Source Generator Design

**Status**: ✅ Implemented in `R3Ext.Bindings.SourceGenerator/ViewForGenerator.cs`

## Implementation Notes

The source generator is now complete and working. Key implementation details:

1. **Interface Detection**: Uses `MetadataName` (e.g., `R3Ext.Activation.IViewFor\`1`) instead of `ToDisplayString()` for reliable interface lookup
2. **Fully Qualified Types**: Uses `R3.Observable<R3Ext.Activation.ActivationState>` to avoid conflicts with `Microsoft.Maui.ActivationState`
3. **Explicit Interface**: Activation property is generated as explicit interface implementation to avoid name conflicts
4. **DI Resolution**: Uses `Application.Current?.Handler?.MauiContext?.Services` as fallback when no `IServiceProvider` is passed

## Problem Statement

Currently, `IViewFor<TViewModel>` requires manual boilerplate:

```csharp
public partial class ActivationDemoPage : ContentPage, IViewFor<ActivationDemoViewModel>
{
    public ActivationDemoPage()
    {
        InitializeComponent();

        // Manual: Create/resolve ViewModel
        ViewModel = new ActivationDemoViewModel();

        // Manual: Sync to BindingContext
        BindingContext = ViewModel;
    }

    // Manual: ViewModel property (no sync back from BindingContext)
    public ActivationDemoViewModel? ViewModel { get; set; }

    // Manual: AutoActivateViewModel (could be defaulted)
    public bool AutoActivateViewModel => true;

    // Manual: Activation observable (could be generated)
    Observable<ActivationState> IActivatable.Activation =>
        PageActivationExtensions.GetActivation(this);
}
```

## Desired Developer Experience

```csharp
// Developer writes only this:
public partial class ActivationDemoPage : ContentPage, IViewFor<ActivationDemoViewModel>
{
    public ActivationDemoPage()
    {
        InitializeComponent();

        // WhenActivated just works - ViewModel already resolved from DI
        this.WhenActivated((ref DisposableBag d) =>
        {
            ViewModel.SomeProperty.Subscribe(...).AddTo(ref d);
        });
    }
}
```

## Source Generator Output

The source generator should produce a partial class like:

```csharp
// <auto-generated/>
#nullable enable

using Microsoft.Maui.Controls;
using R3;
using R3Ext.Activation;
using R3Ext.Maui;

namespace R3Ext.SampleApp;

partial class ActivationDemoPage
{
    private ActivationDemoViewModel? _viewModel;
    private bool _syncingBindingContext;

    /// <summary>
    /// Gets or sets the ViewModel. Setting this automatically updates BindingContext.
    /// </summary>
    public ActivationDemoViewModel? ViewModel
    {
        get => _viewModel;
        set
        {
            if (ReferenceEquals(_viewModel, value))
                return;

            _viewModel = value;

            if (!_syncingBindingContext)
            {
                _syncingBindingContext = true;
                try
                {
                    BindingContext = value;
                }
                finally
                {
                    _syncingBindingContext = false;
                }
            }

            OnPropertyChanged(nameof(ViewModel));
        }
    }

    /// <summary>
    /// Gets a value indicating whether to auto-activate the ViewModel.
    /// </summary>
    public bool AutoActivateViewModel => true;

    /// <summary>
    /// Gets the activation observable for this page.
    /// </summary>
    Observable<ActivationState> IActivatable.Activation =>
        PageActivationExtensions.GetActivation(this);

    /// <summary>
    /// Initializes the IViewFor infrastructure. Call from constructor after InitializeComponent().
    /// </summary>
    private void InitializeViewFor(IServiceProvider? serviceProvider = null)
    {
        // Sync BindingContext changes back to ViewModel
        PropertyChanged += (s, e) =>
        {
            if (e.PropertyName == nameof(BindingContext) && !_syncingBindingContext)
            {
                _syncingBindingContext = true;
                try
                {
                    if (BindingContext is ActivationDemoViewModel vm)
                    {
                        _viewModel = vm;
                        OnPropertyChanged(nameof(ViewModel));
                    }
                    else if (BindingContext == null)
                    {
                        _viewModel = null;
                        OnPropertyChanged(nameof(ViewModel));
                    }
                }
                finally
                {
                    _syncingBindingContext = false;
                }
            }
        };

        // If ViewModel not set, try to resolve from DI
        if (_viewModel == null && serviceProvider != null)
        {
            ViewModel = serviceProvider.GetService<ActivationDemoViewModel>();
        }
    }
}
```

## DI Integration Options

### Option 1: Constructor Injection (Preferred)

```csharp
// Developer writes:
public partial class MyPage : ContentPage, IViewFor<MyViewModel>
{
    public MyPage(MyViewModel viewModel)
    {
        InitializeComponent();
        ViewModel = viewModel; // Generated property handles BindingContext sync
    }
}

// Registration in MauiProgram.cs:
builder.Services.AddTransient<MyViewModel>();
builder.Services.AddTransient<MyPage>();
```

### Option 2: Auto-Resolution via IServiceProvider

```csharp
// Developer writes:
public partial class MyPage : ContentPage, IViewFor<MyViewModel>
{
    public MyPage(IServiceProvider services)
    {
        InitializeComponent();
        InitializeViewFor(services); // Generated method resolves ViewModel
    }
}
```

### Option 3: Parameterless with App.Current.Services

```csharp
// Developer writes:
public partial class MyPage : ContentPage, IViewFor<MyViewModel>
{
    public MyPage()
    {
        InitializeComponent();
        InitializeViewFor(); // Uses App.Current.Handler.MauiContext.Services
    }
}

// Generated helper:
private void InitializeViewFor()
{
    var services = Application.Current?.Handler?.MauiContext?.Services;
    InitializeViewFor(services);
}
```

## Handling Different Base Types

The source generator must detect the base type and use the correct activation source:

| Base Type     | Activation Source                              |
| ------------- | ---------------------------------------------- |
| `ContentPage` | `PageActivationExtensions.GetActivation(this)` |
| `ContentView` | `ViewActivationExtensions.GetActivation(this)` |
| `Shell`       | Custom shell navigation events                 |
| `TabbedPage`  | Per-tab activation                             |
| Custom base   | Fallback to Loaded/Unloaded                    |

## Detection Pattern

The source generator should trigger when it finds:

```csharp
// Pattern 1: Direct implementation
public partial class MyPage : ContentPage, IViewFor<MyViewModel>

// Pattern 2: Via intermediate base
public partial class MyPage : BasePage, IViewFor<MyViewModel>
where BasePage : ContentPage

// Pattern 3: Multiple view models (advanced)
public partial class MyPage : ContentPage, IViewFor<MainViewModel>, IViewFor<SecondaryViewModel>
```

## AOT Considerations

1. **No reflection** - All type detection at compile time
2. **No dynamic dispatch** - Generate concrete types
3. **No `Activator.CreateInstance`** - Use DI or explicit construction
4. **Trim-safe** - Preserve necessary types via attributes

## Open Questions

1. **Should ViewModel be required or optional?**

    - Currently optional (`TViewModel?`)
    - Could require it and throw if null when activated

2. **How to handle ViewModel changes mid-lifecycle?**

    - Deactivate old VM, activate new VM?
    - Just update bindings?

3. **Should we support `IViewFor<>` without type parameter?**

    - For non-generic base classes
    - Would use `object? ViewModel`

4. **Integration with Shell navigation?**
    - Shell passes parameters differently
    - May need special handling
